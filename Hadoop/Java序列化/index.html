Java序列化

序列化

序列化（Serialization）是将对象的状态信息转换为可以存储或传输的形式的过程。反序列化（Deserialization）则是将已序列化的对象状态信息恢复为对象的过程。序列化机制允许在内存中创建的对象被转换为字节序列，这些字节序列可以保存在磁盘上，或通过网络发送到另一个运行Java虚拟机的进程，并在需要时重新构建对象。

Java序列化

要实现序列化，一个类必须实现java.io.Serializable接口。这个接口是一个标记接口，没有定义任何方法。一旦一个类实现了这个接口，Java虚拟机（JVM）就可以将这个类的对象写入字节流中，或从字节流中恢复出对象。

transient

Java中的transient关键字，transient是短暂的意思。对于transient 修饰的成员变量，在类实例的序列化处理过程中会被忽略。因此，transient变量不会贯穿对象的序列化和反序列化，生命周期仅存于调用者的内存中而不会写到磁盘里持久化。
