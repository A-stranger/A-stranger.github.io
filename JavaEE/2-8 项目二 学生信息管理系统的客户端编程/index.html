【实训2-8】项目二 学生信息管理系统的客户端编程

实训目的

了解一个网站静态资源的结构和组成部分

实训内容介绍

  • 创建项目
  • 项目结构
  • HTML 文件
  • CSS 文件
  • JavaScript 文件
  • 实训总结

创建项目

一个项目就是一个应用程序。 创建一个名为student 的动态Web项目（Dynamic Web Project）。

项目结构

按照教材第 39 页图2-7的要求，在 WebContent 下创建 css, images, js文件夹。

HTML 文件

按照教材第 39~45 页的讲解，创建下述 4 个 HTML 文件。

1）登录页面（index.html）：查看并复制参考代码，保存后，预览效果http://127.0.0.1:8080/student/index.html。 这时预览的效果并不好，这是因为缺少两项资源：图片、CSS样式，将在下一步完成。

2）浏览学生信息页面（view.html）：查看并复制参考代码，保存后，预览效果http://127.0.0.1:8080/student/view.html。

3）添加学生信息页面（add.html）：查看并复制参考代码，保存后，预览效果http://127.0.0.1:8080/student/add.html。

4）更新学生信息页面（update.html）：查看并复制参考代码，保存后，预览效果http://127.0.0.1:8080/student/update.html， 这个页面与 add.html 基本相同，不同的地方除了提示文字以外，关键是多了一个表单元素 它的作用是传递当前修改行的主键值。 。

CSS 文件

CSS 的作用是渲染页面外观的，下载图片 header.png文件到 images 目录，或者自行准备一张 800*450 大小的背景图片放到images目录。

![header](img/2-8 项目二 学生信息管理系统的客户端编程/header.png)

按照教材第 45~51 页的讲解，在 css 目录下创建下述 3 个 CSS 文件。

1）公共样式（common.css）

2）登录页面样式（login.css）

3）主页面样式（view.css）

完成后，再预览效果http://127.0.0.1:8080/student/index.html，这时可以看到预期的效果。因为缺少服务器端编程，所以是没有功能的。

JavaScript 文件

按照教材第 51~52 页的讲解，在 js 目录下创建下述 1 个 JavaScript 文件。

1）脚本文件（script.js）：查看并复制参考代码。

完成后，访问添加页面http://127.0.0.1:8080/student/add.html或更新页面http://127.0.0.1:8080/student/update.html， 将会对表单进行校验，例如直接点击“添加”按钮时，会弹出提示信息“用户名不能为空”。

实训总结

本次实训体验了学生信息管理系统的客户端编程，更好地理解 html, css 和 JavaScript 的作用。
