文件系统

文件系统是对文件存储设备的空间进行组织和分配，负责文件存储并对存入的文件进行保护和检索的系统。具体地说，它负责为用户建立文件，存入、读出、修改、转储文件，控制文件的存取，当用户不再使用时撤销文件等。

Windows： NTFS、FAT32（存放的单个文件不能超过4GB）

Linux：ext2、ext3、ext4、xfs

Hadoop：hdfs

HDFS用户

  • 超级用户

If you started the NameNode, then you are the super-user.

  • 用户身份

The identity of a client process is determined by the host operating system. On Unix-like systems, the user name is the equivalent of whoami.

https://hadoop.apache.org/docs/current/hadoop-project-dist/hadoop-hdfs/HdfsPermissionsGuide.html

HDFS Shell

命令格式

HDFS Shell命令是由一系列类似Linux Shell的命令组成的。命令格式如下所示：

hdfs [--config confdir] [--loglevel loglevel] COMMAND

举例

hdfs version

hdfs namenode -format

hdfs dfs -ls /

hdfs dfsadmin -safemode enter

文件扩展属性

设置扩展属性

-n：属性名，要以user/trusted/security/system/raw。比如 user.from

-v: 属性值

-x: 删除文件的扩展属性

获取扩展属性

-d : 显示所有属性

-e: encoding ,属性值以指定的编码输出。text / hex / base64

管理命令

hdfs dfsadmin [options]

run a DFS admin client

https://hadoop.apache.org/docs/stable/hadoop-project-dist/hadoop-hdfs/HDFSCommands.html#Administration_Commands

启用回收站

property default comment
fs.trash.interval 0 保留时间（默认关闭，单位分钟）
fs.trash.checkpoint.interval 0 垃圾回收的检查时间间隔，一般小于保留时间

回收站位置：/user/$USER/.Trash/Current/

对应配置文件：

core-site.xml

安全模式

安全模式下，HDFS只接收读取请求，不接收修改、删除请求。

配置

property default comment
dfs.namenode.safemode.threshold-pct 0.999f 默认副本率= 实际副本数/设置的副本数
dfs.namenode.safemode.min.datanodes 0
dfs.namenode.safemode.extension 30000 默认离开安全模式时间

对应配置文件：

hdfs-site.xml

使用

hdfs dfsamin

-safemode 参数 comment
enter 进入安全模式
leave 强制退出安全模式
get 查看当前状态
wait 等待从安全模式退出
